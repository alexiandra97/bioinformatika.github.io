FORMATI FAJLOVA

FASTA

U bioinformatici FASTA format je tekstualni format nukleotidne ili aminokiselinske sekvence, u kojoj je svaka aminokiselina/nukleotid predstavljena jednim slovom. FASTA format dozvoljava ime sekvence i komentar pre same sekvence (“>…………”). Na početku fajla nalazi se znak veće od “>” praćeni opisom sekvence u jednoj liniji.

Npr.

>MCHU - Calmodulin - Human, rabbit, bovine, rat, and chicken
MADQLTEEQIAEFKEAFSLFDKDGDGTITTKELGTVMRSLGQNPTEAELQDMINEVDADGNGTID
FPEFLTMMARKMKDTDSEEEIREAFRVFDKDGNGYISAAELRHVMTNLGEKLTDEEVDEMIREA
DIDGDGQVNYEEFVQMMTAK*

Literatura:

https://en.wikipedia.org/wiki/FASTA_format

FASTQ

FASTQ format je tekstualni format u kome je sačuvana biološka sekvenca (uglavnom nukleotidna) i njihove odgovarajuće ocene kvaliteta (u ASCII formatu).
FASTQ format ima 4 odvojena polja po sekvenci:

  1. Počinje sa “@” , zatim identifikator sekvence i opis sekvence
  2. Slova sekvence
  3. Počinje sa “+” i ponekad se nalazi opis u nastavku
  4. Ocene kvaliteta u ASCII formatu za sekvencu u polju 2 (svaki znak je kvalitet jednog slova sekvence)
Npr. Ilumina (polje 1)
@HWUSI-EAS100R:6:73:941:1973#0/1

HWUSI-EAS100R Jedinstveno ime instrumenta
6 Flowcell lane
73 Broj pločice u flowcell lane
941 x koordinata klastera na pločici
1973 y koordinata klastera na pločici
#0 indeks broj
/1 Član para reads (ako je paired end (/1, /2))

Literatura:

https://en.wikipedia.org/wiki/FASTQ_format

SAM (Sequence aligment map)

SAM je tekstualni format u kome se čuva biološka sekvenca poravnata na referentnu sekvencu.
SAM format sadrži header i deo sa poravnatom sekvencom. SAM fajlovi mogu biti analizirani i editovani softverom kao što je SAMtools.
Header se nalazi pre poravnatih nukleotida i sadrži 11 obaveznih kolona:

Polje Tip Opis
QNAME String Query template NAME
FLAG Int bitwise FLAGS*
RNAME String Ime referentne sekvence
POS Int 1- based leftmost mapping POSSITION
MAPQ Int MAPping Quality
CIGAR String CIGAR string
RNEXT String Ref. name of the mate/next read
PNEXT Int Position of the mate/next read
TLEN Int observed Template LENgth
SEQ String segment SEQuence
QUAL String Phred-scaled basee Quality+33

Literatura:

https://en.wikipedia.org/wiki/SAM_(file_format)

BAM (Binary Aligment map)

BAM format je binarni ekvivalent SAM formata, koji sadrži iste podatke kompresovane kao binarni zapis.

VCF (The Variant call format)

Header: meta-information lines; redovi pocinju sa # i sadrze standardizovan opis tagova i anotacija koje su koriscene u dokumentu;
tu se takodje mogu naci podaci o prethodnoj obradi (da bi se stiglo do vcf fajla) i referentonom genomu.
8 obaveznih kolona sa informacijama:

  1. Chromosome (CHROM)
  2. pozicija (POS)
  3. jedinstvena identifikacija varijante (ID)
  4. referentni alel (REF)
  5. alternativni nereferentni aleli (ALT)
  6. phred-scaled kvalitet (QUAL)
  7. mesto filtriranja info (FILTER)
  8. dodatne informacije (INFO)
FORMAT npr. GT:GQ:DP – genotip, kvalitet genotipa i dubina citanja svakog uzorka

BED (The browser extensible data format)

BED format je tekstualni format koji sadrži genomske regione kao kordinate i povezane anotacije. Podaci su predstavljeni u formi kolona odvojenih razmacima i tabovima.
Prednost ovog formata je lakša i brža manipualcija podacima pomoću skriptnih jezika.
BED fajl sadrži najmanje tri kolone, i devet opcionih
Literatura:

https://en.wikipedia.org/wiki/BED_(file_format)
https://www.ensembl.org/info/website/upload/bed.html
